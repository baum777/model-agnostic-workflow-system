# Promotion Policy

Class: operational.
Use rule: memory can become canonical only through explicit review.

## Rule

Memory does not automatically promote.

Promotion requires:

- explicit review
- named reviewer
- named target
- source memory ID
- evidence path or provenance

## Allowed Targets

- `SOT.md`
- `DECISIONS.md`
- `MEMORY.md`
- `docs/*`
- `core/contracts/*`

## Phase 2 Boundary

This policy defines the review boundary only. No promotion command exists in Phase 2.
